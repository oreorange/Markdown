# マネジメント

## 6. 開発技術
```
用語・試験頻出
- システム要件の種類　機能要件（ユーザーへのヒアリング）と非機能要件（応答時間やセキュリティ性）
- 品質特性　（機能性、信頼性、使用性、効率性、保守性、移植性）の6つがある
- システム方式設計　システム要件を（ハードウェア、ソフトウェア、手作業）に振り分ける
- ソフトウェア要件定義　ソフトウェアの要件（機能や性能）を決める。主にインターフェースとデータ
- ソフトウェア方式設計　ソフトウェア要件（機能や性能）をプログラムの単位まで分割する
- ソフトウェア詳細設計　プログラムの単位まで分割された要件を、さらにコーティングができる単位まで分割する
- テスト（システムテスト、結合テスト、ブラックボックステスト、ホワイトボックステスト、ソフトウェア受入れ）V字の図を覚えよう
- インタフェース　あるモノとあるモノを繋ぐ部分のこと。ユーザインタフェース　コンピュータと人間を繋ぐ。
- ファンクションポイント法　システムの機能に点数をつけて、開発費用を見積もる方法
- アジャイル(素早い)開発　短期間にソフトウェアの開発とリリースを繰り返し、ビジネス環境の変化やユーザのニーズに柔軟に対応する開発モデル
- リバースエンジニアリング　既にあるプログラムを解析して、仕様や設計情報を取り出す技術
```

## 7. プロジェクトマネジメント
```
用語・試験頻出
- プロジェクト 期限内に独自のモノやサービスを作るための業務。
- PMBOK Project Management Body of Knowledge プロジェクトマネージャーのための教科書
- ガンチャート　プロジェクトの進捗状況を視覚的に表す図
- アローダイアグラム 作業の流れを矢印で表した図
- クリティカルパス プロジェクトにおいて最も時間のかかる経路
- プロジェクト資源マネジメント プロジェクトに参加する要因の役割、責任、必要なスキルを決定し、効果的に機能できるよう管理すること
- プロジェクトコミュニケーションマネジメント 誰が、誰に、いつ、どうやって、何を伝達するかを管理
- プロジェクトリスクマネジメント リスクの対策案を管理し、問題が発生した場合には対応すること
```
